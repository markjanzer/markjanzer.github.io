Destructive JavaScript Array Methods

October 29th, 2015

I have had trouble determining which JavaScript methods are destructive and which are not. Especially coming from Ruby, it is rather jarring to see that methods like concat are not destructive. I am going to make a list of the destructive array methods.

The List

  • .copyWithin(target, start, [end=array.length])
  • .fill(value, [start=0], [end=this.length])
  • **.forEach(callback(element, index, array))**
  • .pop()
  • .push(element1, [...elementN])
  • .shift()
  • .sort([compareFunction])
  • .splice(start, deleteCount, [item1], [...itemN])
  • .unshift(element1, [...elementN])

How to Use

.copyWithin


  array.copyWithin(target, start, [end=array.length])
This method reads the contents at an array starting at the index of start (the second parameter), and stops reading either at the end of the array or at the specified end (which is not inclusive, hense the default to array.length instead of array.length-1). It then copies the elements it just read into the array starting at the target index, replacing all the values.

In this array, the copyWithin statement takes the values at indexes 4 and 5 (values 4 and 5), and copies them over the array starting at the index 1, thus replacing 1 and 2. 


  var arr = [0,1,2,3,4,5,6];
  arr.copyWithin(1,4,6);
  console.log(arr); //=> [0,4,5,3,4,5,6]

.fill


  array.fill(value, [start=0], [end=this.length])
This replaces elements in an array with the value given to fill. You can select a start index or a (non-inclusive) ending index.

.forEach


  array.forEach(callback(element, index, array))
This function is not necesarily destructive, but I put it in here because it can be destructive if you use it to change the values of array[index] with the callback function. Below is an example of how this works. 

  function doubleArrayElements(element, index, array) {
  array[index] = element * 2;
  }

  var arr = [1,2,3,4];
  arr.forEach(doubleArrayElements);
  console.log(arr); //=> [2,4,6,8]

.pop


  array.pop()
This method removes the last element from the array. To capture that value before it leaves the array and disappears, the .pop() function also returns that "popped" value. 
  var arr = [0,1,2,3,4]
  var lastValue = arr.pop();
  console.log(arr); //=> [0,1,2,3]
  console.log(lastValue) //=> 4

.push


  array.push(element1, [...elementN])
.push adds an element to the end of an array. If you have multiple values to push to the end of the array, .push will add them in the same order that you input them.

.shift


  array.shift()
.shift() does the same thing as .pop(), it just removes the first element rather than the last. .shift() also returns the element.

.sort


  array.sort([compareFunction])
.sort automatically sorts your array. The default sort does so in this order: numbers ascending, followed by upper case alphabetical, followed by lower case alphabetical. Only the first character is evaluated in strings. 

  var arr = ["A", "a", 0, 100, "Zebra", "z", " space first"];
  arr.sort();
  console.log(arr);
  //=> [ ' space first', 0, 100, 'A', 'Zebra', 'a', 'z' ]

.splice


  array.splice(start, deleteCount, [item1], [...itemN])
The .splice method removes values from a specific portion of the array and optionally adds values to the same location of the array. You have to specify the index where you want to start deleting items, and the number of items to delete. You can also use this function to not remove any items, but add new values in at the start index. 
  var arr = [0,1,2,3,4,5,6]
  arr.splice(4, 2, "four", "five");
  console.log(arr);
  //=> [0,1,2,3,"four","five",6]
  arr.splice(2, 1, "sup", "brew");
  console.log(arr);
  //=> [ 0, 1, 'sup', 'brew', 3, "four", "five", 6 ]

.unshift


  array.unshift(element1, [...elementN])
The .unshift method adds elements to the front of the array, shifting all of the current elements back indexes.

I hope this helps! Good luck JavaScriptologists!
