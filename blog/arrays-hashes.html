Arrays and Hashes

October 4th, 2015

Arrays and hashes are two of the more exciting objects in Ruby. They are both collections of other objects, stored within the array or hash, and paired to an index. These indexes are the keys to finding the values appropriate values in the array/hash. Both of these awesome objects are similar, but they each have their unique differences making them useful in different situations.

Arrays are ordered lists of values, and by adding these values to the array, they are automatically assigned an index value, indicating their placement in the array. 


    coolest_to_least = [“Me”, “The Rock”, 8, false, “food poisoning”]

If I wanted to find the coolest thing in my array, I could ask for the first element by coolest_to_least[0], and it would tell me that “Me” is the first thing in the cool list. I did not assign this index 0 to the value “Me,” but the index was automatically attached to the value as I defined the array.

Hashes are unordered collections of key value pairs. Unlike arrays (which keep track of order and automatically assign indexes), hashes require you to define your own indexes. This allows for lists with with more descriptive keys, and is useful in different situations. 


    personal_info = {
    firstName: “Mark”
    lastName: “Janzer”
    favoriteNumber: 8
    SSN: “Hahahaha”
    }

If I had put this information in an array, it would show up as such [“Mark”, “Janzer”, 8, “Hahahaha”] This information is seemingly meaningless and arbitrarily ordered. The hash makes more sense and instead of using a number such as [0] to call my first name, I can do personal_info[firstName].

Arrays are useful ordered lists of information that don’t require unique tags, while hashes are useful for grouping of information that has unique labels and doesn’t require order.
