CSS Positioning

Steptember 26th, 2015

I am going to address the differences between static, relative, absolute, and fixed positioning. Static positioning is the default positioning of html elements. These elements follow the standard flow of the document. The only reason why you would declare static positioning would be to override previously stated positioning.

When an element has relative positioning, it can be moved using top, bottom, left, or right. Let’s say I have a class “awesomeness” that I set to relative positioning and move that class right 20 pixels. But 20 pixels from where?

With relative positioning elements are moved relative to their static position (or where they would be if their position was static).

Finally, it is important to know that a relative positioning keeps the element in the flow of the document. This is not true for all positions.

When an object has absolute positioning, it is no longer a part of the flow of the document. Thus it is easy to accidentally overlap with other elements. While static elements naturally avoid each other, they will not avoid an object with absolute positioning.

When you move an element with absolute positioning, where is it moved in relation to? It is moved from the closest parent element that has relative or absolute positioning. If there is no such parent, it will move in relation to the HTML document as a whole.

Fixed positioning can be thought of as a type of absolute positioning. Both fixed and absolute positioning remove the element from the flow of the page. While absolute positioning uses a parent element as reference, fixed positioning uses the browser window as reference. Thus, no matter where you scroll on the page, the element with fixed positioning will always appear at the same location.
