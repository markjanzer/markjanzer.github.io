The Select Method

October 11th, 2015

Imagine you are given an array of ten random numbers between 1 and 1000, and you are asked to return an array only containing those numbers divisible by seven. You could go about solving it using the each method: 


    divisible = []
     rand_nums.each do |n|
       if n % 3 == 0
       divisible.push(n)
      end
    end

We are going through each element of the rand_nums array and checking if the number is divisible by three. If the number is divisible by three, we have to push it to a new array that we make beforehand. There is a better way to do this.

When called on an enumerable object, the .select method returns an object of the same type composed of every value for which the block evaluates to true. 


  rand_nums.select { |n| n % 3 == 0 }
In this example, I call the select method on my rand_nums array with a block following. In the block I evaluate each number (n) to test if n is divisible by three. If this evaluates to true, it will be added to the object that is returned, which will be an array because rand_nums is an array. This code will return an the array of random numbers only with numbers that are divisible by three.

To really get the grasp how .select returns the object, here are two silly examples: 


  any_object.select { |var| true}
This will always return any_object as is. Select will iterate through each entry, see that the block evaluates to true, and return every value of any_object in the new object. 

  any_object.select { |var| false }
This will always return an empty object of the same type of any_object. Select will iterate through each entry and see that the block always passes false, and thus add nothing to the object.
