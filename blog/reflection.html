Website Reflection

November 8th, 2015

Creating my own website has been a great experience for me. It has been both frustrating and rewarding, a fantastic learning experience with a practical result.

One of the largest challenges for me was working with CSS positioning. With programming languages like Ruby or JavaScript, errors make sense. When something goes wrong, I can find out why the error is arising, and the output that I get then makes sense. I am sure that the CSS positioning is logical too, but it is much more difficult for me to see why things are being positioned where.

I guess that one of the difficult things is that I have not found as many tools for CSS to help practice positioning, and it is more difficult to troubleshoot using the web. With a language you can look up multiple different resources to practice writing certain functions. Most of the problems you work on have already been answered, and people have already asked most of your questions. However, nobody has designed my site before, and there could be a wide variety of reasons why I can’t move an image to the right.

Only once I started learn Foundation did I start to see how I had been tricked, just like I had been tricked in math class. I had been taught the hard way before I learned the easy way (which makes complete sense from an educational standpoint). It didn’t really occur to me that CSS could be made so much easier with the use of frameworks. I didn’t know that there would be an easier way. I also started using Emmet, and had a whole world of convinience opened up to me.

This all leaves me with a bit more hope for potentially revamping my website and enjoying it along the way.
