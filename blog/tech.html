JavaScript's Compare Function

October 30th, 2015

I have been confused by JavaScript's sort function long enough. The array.sort() function takes an optional function as an argument in the parentheses, and I have seen this compare function put to great use. It is time to learn how to use this compare function!

By default, the sort function sorts elements in an array in ascending order by their unicode value. For now, we are going to deal with only numbers to make things more simple, otherwise things get tricky when comparing different types of objects with the compare function. Below are compare functions (for numbers) written to be the same as .sort(), the opposite, and to do nothing. 


      var nums = [8,6,5,2,3,9,1];
      nums.sort(function(a,b){
        return 0;
      });
      console.log(nums); //=> [8,6,5,2,3,9,1];
      // This does nothing.

      nums.sort(function(a,b){
        return a - b;
      });
      console.log(nums); //=> [1,2,3,5,6,8,9];
      // This sorts in ascending order.

      nums.sort(function(a,b){
        return b - a;
      });
      console.log(nums); //=> [9,8,6,5,3,2,1];
      // This sorts in descending order.

This might not be how the inner workings of the compare callback function work, but here is what makes sense to me.

This function takes two arguments, a and b. a represents the current element, and b represents the element it is being compared to. The function is run, and a will move or not move depending on what the function returns. If the function returns a positive number, a will be moved an index after b. If a negative number is returned, a will move an index before b. If zero is returned, a will not change it's location. Now how JavaScript chooses which elements to compare to what is beyond me. There are different algorithms for sorting, some of which can be found here.

Now let's try something cool. Lets say that three kids are trying to find out who knows the longest word. They have expansive vocabularies, but aren't that great at counting, so they put their words in their own objects and hope that their friendly programmer can figure out the answer. Let's give it a shot. 


     var students = [
     {name: "Jack", crush: "Cindy", largestWord: "gasconading"},
     {name: "Cindy", crush: "Jessie", largestWord: "sesquipedalian"},
     {name: "Jessie", crush: "Johnny Depp", largestWord: "osculator"}
     ];

     // Now to sort the students in Descending order of longest word.
     students.sort(function(a,b){
      return b.largestWord.length - a.largestWord.length;
     })

     console.log(students[0].name, students[1].name, students[2].name);
     //=> Cindy Jack Jessie
