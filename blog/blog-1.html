Intro to Git and GitHub

September 11th, 2015

What is Version Control

Let's say I am a sculptor and I am working on a massive marble sculpture of myself. I am chipping away at this marble, shaping a far more muscular version of myself. But something goes wrong. I back up and realize that my biceps are way too small. It's a disaster. If I had version control, I could revert the statue back to a time when I had not yet sculpted the arms. I could even revert the stone to it's original giant slab of marble. I could go back and control whichever version of the project I wanted. But I don't have version control, because stone is not digital, and I am left to wallow in my own unique concoction of vanity and sorrow.

If you have worked with any aspect of Google Drive, newer Microsoft Word, or certain computer back up services such as Time Machine or Back Blaze, you have worked with Version Control. Version control systems allow you to go back in time to previous versions and use them as you wish, whether it is copy and pasting as section, or restoring the whole version.

Another feature some version control offers is that of branches. It would be as if I as a sculptor entered a parallel universe, and decided to promote realistic body standards and chiselled a body that more accurately represented my own. Then I could decide that this parallel universe's statue is superior, and replace the first universe's statue with this less conceited one.

Let me put it in realistic terms. Lets say we are making a website to save the world. You are working on the world saving algorithm, and I am working on making it pretty with CSS and HTML.We can each make different branches (copies of the project) and work on them independently. Then, we can each use our branch to update the master branch when we are done with our equally important jobs.

Git and GitHub

Let me make this clear. Git is a version control system. You run it locally, and it allows you to do everything we discussed earlier. GitHub is a digital space that allows people to use git alongside other people. Git is the VCS (Version Control System). GitHub is a place for people to share and use git together. I make this point because I was confused about this for quite a while.

I'm going to go into a little more detail about git without getting too technical. Git allows you to create and work on different branches of a project so you don't have to change the original while editing. Later, you can merge the branch with the master branch to change the original. You can also save your progress using commits. If you find that you wanted a previous commit, you can type "git log," find the commit that you accurately labeled and your files to their former greatness.

GitHub allows you to do all of this with other people over different computers, at different times. You have a shared repository with it's own master branch and revision history. You can all work simultaneously and merge your work when convinient. Even if you are working by yourself, GitHub is a good place to save your work. If something were to happen to your computer, your files are safe because they are now stored in the cloud.

Git is an amazing tool and GitHub is a magical place. I hope to teach with more depth in a later post. Until then, happy coding. Or sculpting.
