Ruby Classes vs JavaScript Constructors

October 23rd, 2015

When Rubyists need to create multiple similar objects, they define a Class. When JavaScriptorians (cool word huh?) need to create multiple similar objects they use a Constructor. To explore these differences, let’s make a Ruby class, and then attempt to replicate it using JavaScript constructors. 


    class Beverage
      def initialize(name, size_in_oz)
       @name = name
        @remaining = size_in_oz
    end

     attr_reader :remaining

     def sip
        @remaining -= 1
      end

    end

Now to make this in a constructor! The constructor itself is very similar to defining a function. To set properties to the object, you use the “this” keyword. 


    function Beverage(name, size_in_oz) {
      this.name = name;
      this.remaining = size_in_oz;
    }

    var rogueBeer = new Beverage("Dead Guy Ale", 12)

This is working out! The constructor uses similar syntax to Ruby’s initialize, taking the variables as arguments, and then defining them within the body. Instead of @name to set name as an instance variable, we are using this.name to set name as a property of the Beverage object. Also, by defining this.name = name; we already have a reader attribute for the value name.

Things start to get tricky now. You can throw your functions right into the function brackets as such: 


    function Beverage(name, size_in_oz) {
      this.name = name;
      this.remaining = size_in_oz;
      this.sip = function(){
        this.remaining -= 1;
        return this.remaining + " ounces are remaining."
      }
    }

However you also can use prototypes to store the functions and variables. In JavaScript it, it seems like it is best practice to do this with any properties that would not be in the initialize portion of a Ruby class. Here is how it would be done: 


    function Beverage(name, size_in_oz) {
      this.name = name;
      this.remaining = size_in_oz;
      this.sip = function(){
        this.remaining -= 1;
        return this.remaining + " ounces are remaining."
      }
    }
    Beverage.prototype.sip = function() {
      this.remaining -= 1;
      return this.remaining + " ounces are remaining."
    }

I believe that using prototypes is best practice due to how efficiency and how inheritance works.
